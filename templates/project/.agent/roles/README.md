# Repo-Local Specialist Role Specs

These files are the portable role contract generated by Shrey Junior.

Use them with this precedence:

1. existing trusted repo authority files
2. promoted repo-local canonical docs explicitly referenced by that authority
3. `.agent/project.yaml` and `.agent/checks.yaml`
4. these specialist role specs
5. generated task packets and continuity artifacts

What these files are for:

- give local and cloud-visible agents a stable repo-local description of specialist purpose
- keep specialist guidance portable without relying on `~/.codex`, `~/.claude`, or editor-local prompts
- define validation expectations, MCP eligibility, and handoff guidance in a human-readable form

What these files are not:

- they are not stronger than explicit repo authority
- they are not proof that a tool runtime will obey every instruction
- they do not replace CI or push/release checks

Generated from:

- `configs/specialists.yaml`
- managed by `shrey-junior bootstrap`, `sync`, or `standardize`
