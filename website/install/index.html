Desktop install

1. Download the macOS pkg from the homepage or all downloads.
2. macOS: the pkg installer is the intended default path for install-time kc setup.
3. Windows: EXE/MSI desktop installers are not live yet. Use the PowerShell CLI bootstrap for kc.
4. Choose a repo, initialize it if needed, and keep using desktop or kc after the platform-specific setup succeeds.

Default terminal command setup

1. macOS pkg install is the intended default path for install-time CLI setup.
2. Windows setup EXE is the intended default Windows path once it exists, but proof is still pending on a real Windows host.
3. Current unsigned macOS beta builds may still need a manual first-open override after install until Developer ID signing and notarization are configured.
4. If Windows desktop setup is blocked, use the wrapper CLI path below. Keep Windows desktop unavailable until EXE/MSI are published.

macOS unsigned beta bypass

1. If Finder says Apple could not verify the pkg, locate the downloaded .pkg in Finder.
2. Control-click the pkg and choose Open.
3. If macOS still blocks it, open System Settings > Privacy & Security and click Open Anyway.
4. Only do this for the unsigned beta downloaded from the Kiwi Control site or after you verify its checksum.

Gatekeeper help

What the macOS warning actually means

This beta warning means the pkg is unsigned or not notarized yet. It does not mean Kiwi Control proved the installer is trusted. Treat it as an unsigned technical-user beta and verify the checksum before bypassing Gatekeeper.

Manual assets

Need DMG, the macOS app tarball, MSI, or CLI bundles?

Use all downloads for the full public asset list. The downloads page also exposes checksums, the release manifest, and the standalone CLI bundles.

The standalone CLI bundle installs kiwi-control and kc only. It does not install the desktop app. If Kiwi Control Desktop is already installed, kc ui can launch or attach to it.
