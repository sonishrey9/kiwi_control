# Roadmap

## Public beta

- publish CLI artifacts to GitHub Releases
- wire `sj-ui` to a real local backend path powered by `sj-core`
- finalize Homebrew and winget publication
- improve iOS and Android repo detection without widening generic defaults

## 1.0 readiness

- signed desktop bundles on macOS, Windows, and Linux
- stable updater metadata flow
- broader package-manager documentation
- stronger migration helpers for older repo contracts

## Open-core direction

Likely open-source core:

- repo contract generation
- routing
- checkpoints
- handoffs
- memory bank
- CLI

Likely future monetization areas:

- premium desktop workflows
- team reporting and policy packs
- enterprise release and governance integrations

Any monetization path should preserve:

- local-first core operation
- repo-local authority
- honest runtime capability boundaries
